# BeUniq Style Profiles

Use these profiles when the user wants to choose from the BeUniq base instead of providing a custom reference. A selected profile is a starting point; the target repo's tokens and component collection remain the implementation source of truth.

## beuniq-minimal-productive

- Best for: dashboards, internal tools, B2B SaaS, expert workflows.
- Buttons: compact rectangular buttons, 8px radius, solid primary, quiet secondary, clear disabled/loading states.
- Fields: solid or lightly tinted fields, visible border, strong focus ring, inline error text.
- Cards: restrained bordered panels, 8px radius, low or no shadow, dense content hierarchy.
- Modals: focused task dialogs with direct primary/secondary actions.
- Avoid: glow, glass, decorative gradients, oversized hero spacing.

## beuniq-linear-saas

- Best for: product websites, developer tools, polished SaaS apps.
- Buttons: crisp medium-density buttons, 8-10px radius, neutral secondary, one accent primary.
- Fields: clean bordered controls with compact labels and helper text.
- Cards: subtle surfaces, thin borders, measured spacing, strong text hierarchy.
- Modals: simple centered dialogs, restrained overlay, predictable actions.
- Avoid: purple-blue gradient dominance, fake metrics, generic feature-card repetition.

## beuniq-apple-native

- Best for: consumer apps, mobile-first product surfaces, premium utility tools.
- Buttons: native-feeling controls, rounded but not pill-heavy, strong touch targets.
- Fields: platform-like inputs, clear focus/validation, calm spacing.
- Cards: soft grouped surfaces, modest radius, strong platform restraint.
- Modals: sheets/dialogs with crisp motion and clear dismiss behavior.
- Avoid: web-template hero effects, center-origin popovers, transition-all motion.

## beuniq-editorial

- Best for: content-led brands, portfolios, media, launches with strong writing.
- Buttons: minimal command buttons, text+icon links, restrained primary action.
- Fields: simple newsletter/contact fields with clear labels.
- Cards: asymmetric content blocks, image/text rhythm, fewer repeated equal cards.
- Modals: rare; prefer inline flows unless task focus requires a dialog.
- Avoid: generic SaaS cards, fake social proof, vague transformation copy.

## beuniq-premium

- Best for: high-trust services, luxury, finance, health, legal, boutique products.
- Buttons: deliberate spacing, quiet contrast, subtle border/solid variants.
- Fields: high-legibility forms, explicit labels, calm validation states.
- Cards: composed sections, restrained elevation, careful typography.
- Modals: minimal interruption, high clarity, no decorative celebration effects.
- Avoid: fake premium words, glossy effects, unsupported claims.

## beuniq-playful

- Best for: creative tools, education, community products, lightweight consumer apps.
- Buttons: friendly shape, clear active feedback, one expressive accent.
- Fields: approachable controls, helpful empty/error states.
- Cards: varied rhythm, purposeful illustration/media, not repeated template tiles.
- Modals: approachable copy, clear action hierarchy.
- Avoid: unearned confetti, constant float/pulse, animation without reduced-motion.

## custom

Use when the user provides a reference, brand rules, or a specific component collection style that should override the BeUniq base. Record the reference and overrides in `DESIGN.md`.
